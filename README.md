# pages-on-wc2026cn

本仓库用于归档与发布多个独立的 HTML 页面。这些页面彼此之间没有强关联，也不隶属于任何特定域名或网站，仅作为静态 HTML 文件的集中存储与展示用途。

## 项目简介

这是一个功能简单的 HTML 页面集合仓库。每个页面独立存在，可单独访问，无需后端支持。适合存放各类轻量级文档、信息页、工具界面或其他不需要复杂交互的静态内容。

## 目录结构说明

仓库根目录下直接放置 HTML 文件，或按需创建子目录进行归类。典型的目录结构如下：

```
/
├── index.html          # 仓库入口或导航页（可选）
├── page-a/
│   └── index.html      # 一个独立的页面
├── page-b/
│   ├── index.html
│   └── style.css       # 页面相关资源
└── archive/
    ├── old-page.html   # 归档的旧版本页面
    └── ...
```

- 每个文件夹通常对应一个独立的页面或一组相关页面。
- 静态资源（CSS、JavaScript、图片等）建议与 HTML 文件放在同一目录下，或使用相对路径引用。

## 页面归档说明

- **归档目的**：保存不同时期创建的 HTML 页面，便于回溯与访问。
- **版本管理**：若同一页面有多个版本，建议以 `v1`、`v2` 或日期后缀（如 `2024-01-01`）区分，并放入对应的子目录。
- **访问方式**：通过 GitHub Pages 或其他静态托管服务，可直接访问仓库中的 HTML 文件。无需额外配置，将仓库设置为 Pages 源即可。

## 维护说明

- **添加新页面**：直接创建新的 HTML 文件或文件夹，并确保引用资源路径正确。
- **更新已有页面**：建议保留旧版本（归档），新建文件或使用版本号区分，避免覆盖。
- **清理与归档**：过时或不再使用的页面可移入 `archive/` 目录，或直接删除。
- **贡献**：本仓库目前为个人维护，不接受外部直接推送。如有建议，欢迎通过 Issues 讨论。
